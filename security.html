🔑

APIキーの一元管理・分離

✓ 実施済み

Notion・Claude API・LINE Messaging APIのトークンが各プロジェクトフォルダの .envファイルに分散・平文保存されていた状態を改善。 共通APIキーを単一の安全な場所に集約しました。

実施内容
  • ホームディレクトリ直下に .secrets/.env を作成し、共通キーを集約
  • 各プロジェクトの .env はDB IDなどプロジェクト固有値のみに限定
  • Pythonスクリプト3本・Node.jsサーバーのenv読み込み処理を更新（共通→プロジェクト固有の優先順位）
  • .secrets フォルダをWindowsの隠しフォルダに設定
効果
  • プロジェクトフォルダを第三者と共有・コピーしてもAPIキーが漏洩しない
  • キーの棚卸し・ローテーションが1か所で完結する
🔒

シークレットファイルのアクセス権限制限

✓ 実施済み

.secrets フォルダのWindowsアクセス権限を自分のユーザーアカウントのみに制限。 他のユーザーアカウントや一般的なプロセスからの読み取りを防止します。

実施内容
  • icacls コマンドで fkdka ユーザーのみフルコントロールに設定
  • attrib +h で隠し属性を付与（エクスプローラーから見えない）
🔐

ダッシュボードへのBasic認証追加

✓ 実施済み

localhost:3456 で動作するダッシュボードが認証なしで操作できた問題を修正。 ブラウザの標準的なBasic認証ダイアログで保護します。

実施内容
  • server.js に checkAuth() 関数を実装
  • 認証情報は .secrets/.env の DASHBOARD_USER / DASHBOARD_PASS で管理
  • 認証失敗時は 401 Unauthorized を返し、ブラウザが認証ダイアログを表示
  • OPTIONSリクエスト（CORSプリフライト）は認証スキップ（仕様上の必要性）
効果
  • 同一LAN内の他端末からダッシュボードを操作されるリスクを防止
  • ツールの起動・停止・設定変更を認証者のみに限定
🌐

CORSポリシーをlocalhostのみに制限

✓ 実施済み

APIサーバーが Access-Control-Allow-Origin: *（全オリジン許可）で動作していた問題を修正。 悪意あるウェブサイトがユーザーのブラウザ経由でAPIを呼び出せる状態（CSRF的リスク）を解消します。

変更内容
項目 変更前 変更後
許可オリジン * （すべて許可） localhost / 127.0.0.1 のみ
file:// ページ 許可 引き続き許可（null オリジン対応）
外部ドメイン 許可 ブロック
🔌

サーバーを127.0.0.1にバインド（外部アクセス完全遮断）

✓ 実施済み

Node.jsサーバーがデフォルト（0.0.0.0）で起動していたため、 同一LAN内の端末から直接アクセス可能な状態でした。 127.0.0.1 にバインドすることでOSレベルで外部アクセスを遮断します。

実施内容
  • server.listen(PORT) → server.listen(PORT, '127.0.0.1') に変更
  • 管理者権限不要・OS設定変更なしで即時有効
  • ファイアウォールより確実（Windowsファイアウォールは設定が上書きされる可能性がある）
効果
  • カフェや公共Wi-Fiでも外部からのアクセスが物理的に不可能
  • Basic認証との2段構えで、より堅牢な防御を実現
📝

server.js内のハードコードトークンを環境変数化

✓ 実施済み

server.js の /sns-approved-posts エンドポイントにNotionトークンが ソースコード内に直書きされていた問題を修正。

実施内容
  • const NOTION_TOKEN = 'ntn_...' → process.env.NOTION_TOKEN に変更
  • 起動時に .secrets/.env から自動読み込み

⚠️ 残課題（定期メンテナンス）

依存パッケージの脆弱性チェック

月1回程度 npm audit / pip-audit を実行し、既知の脆弱性がないか確認する。

Xプロファイルフォルダの権限制限

sns-generator/x_profile/ にはX（Twitter）ログインセッションが保存されている。.secrets と同様にアクセス権限を制限することを検討。

APIキーの定期ローテーション

Notion・Claude・LINEの各APIキーを3〜6か月ごとに再発行し、.secrets/.env を更新する。

Playwrightの--no-sandboxフラグ見直し

スクレイピングスクリプトがChromiumサンドボックスを無効化して起動している。スクレイピング先が信頼できる既知サイトのみであれば許容範囲内。
