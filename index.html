SNS投稿 自動生成ツール

〜 AIトレンドをリサーチして、投稿案3件をNotionに自動保存 〜

🤖 Claude API / Claude Code

▲ X（Twitter）スクレイピング → パターン分析 → Claude APIでオリジナル生成 → Notion DB保存

「SNS投稿のネタが思いつかない」「毎日投稿したいけど時間がない」——そんな課題を解決するために作ったのがSNS投稿自動生成ツールです。X（Twitter）から高エンゲージメント投稿をリアルタイムで収集・分析し、「バズる書き方パターン」だけを抽出。そのパターンをもとにClaude APIが完全オリジナルの投稿案を3パターン生成してNotionに保存します。

① X（Twitter）で高エンゲージメント投稿をスクレイピング
② 書き方パターンのみを抽出（内容・固有名詞は除外）
　　　　↓
② Claude APIがトレンドを分析・投稿案を3件生成
　　　　↓
③ Notion DBに「下書き」として自動保存
　　　　↓
④ Notionで確認・編集 → X（Twitter）に投稿
💡 投稿ネタ考えの時間がゼロに 📈 トレンドに乗った投稿が自動生成 📋 Notionで投稿案を一元管理・蓄積

GSCカニバリゼーション分析ツール

〜 SEOの共食いをビジュアルで一発チェック 〜

🤖 Claude API / Claude Code

▲ GSC APIからデータ取得 → カニバリ自動検出 → 可視化まで5ステップ

SEOでよくある問題が「記事カニバリゼーション」。同じキーワードで複数記事がGoogleに評価され、お互いの順位を下げ合う現象です。GSCのデータを自動取得し、カニバリゼーションを一覧・可視化するHTMLツールを作りました。

操作方法

  1. ブラウザで gsc_cannibalization.html を開く
  2. サービスアカウントキー（JSON）とサイトURLを入力
  3. 期間を指定して「分析開始」をクリック
  4. カニバリゼーションが疑われる記事ペアが一覧表示
⏱️ 数時間の作業が約3分に 📊 統合すべき記事が一目でわかる 📥 CSVエクスポートで施策に活用

Gmail LINE通知ツール

〜 Gmailの全メールをリアルタイムでLINEに転送 〜

🤖 Claude API / Claude Code

▲ Gmail受信 → LINE即時通知（全メール対象）

メールは来るが見逃す。でもすべてのメールをLINE通知するとうるさい。そこで作ったのが、AIが「重要かどうか」を判断してからLINEに通知するツールです。Gmailを60秒ごとに自動チェックし、新着メールをすべてLINEにリアルタイム転送します。見逃しゼロを実現するシンプルな設計です。

操作方法

  1. ダッシュボードの「Gmail LINE通知」をクリック
  2. 初回のみ：ブラウザが開きGmailログインを促される
  3. ログイン後はバックグラウンドで自動実行
  4. 重要メールが来るとLINEに通知が届く
✅ 重要メールの見逃しがほぼゼロに 🚫 スパム・不要メールはスルー 📱 PCなしでもスマホで確認可能

実際に行ったセキュリティ設定

〜 運用して気づいた落とし穴と、実際に施した対策の紹介 〜

🤖 Claude API / Claude Code

ツールを実際に運用していくなかで、セキュリティ面でいくつかの問題点に気づきました。具体的な実装内容はセキュリティ上の理由から非公開ですが、対策した領域と考え方を紹介します。

⚠️ 運用して気づいた失敗パターン
APIキーをコードに混在させていた → フォルダ共有時に漏洩リスクがあった
ローカルサーバーをデフォルト設定で起動 → 意図せず外部から到達できる状態だった
ダッシュボードに認証なし → 同じネットワークの端末から操作できる状態だった

① APIキーをプロジェクト外で一元管理

複数ツールで利用するAPIキーが各プロジェクトフォルダに分散していたため、プロジェクトとは別の安全な場所にまとめて管理する構成に変更しました。

  • プロジェクトフォルダ内には、そのツール固有の設定値のみを残す構成にしています
  • フォルダをコピー・共有してもAPIキーが含まれない状態を実現しています
  • 保存先にはOSレベルのアクセス制限を設けています

② ローカルサーバーの外部アクセスを遮断

ダッシュボードのサーバーがデフォルト設定のままで、同じネットワーク上の別端末からアクセスできる状態でした。サーバーの待ち受け設定を変更し、ローカルからの接続のみ受け付けるよう修正しています。

※ 設定変更はサーバー起動コード1行で実現できます。管理者権限も不要です。

③ ダッシュボードにアクセス認証を追加

起動・停止・設定変更ができるダッシュボードが認証なしで操作できる状態でした。パスワード認証を導入し、認証情報はプロジェクト外で管理する構成にしています。

④ 依存パッケージの脆弱性チェックを定期実施

利用しているライブラリに既知の脆弱性が発見されることがあるため、定期的にチェックするコマンドを実行しています。ツールごとに対応するコマンドが用意されています。

💡 対策のポイント
① APIキーをプロジェクト外で一元管理　② サーバーの外部アクセスを遮断　③ 操作画面に認証を追加　④ パッケージ脆弱性を定期チェック
いずれも「設定ミスや意図しない公開状態」をなくすことが目的で、個人開発レベルのリスクはこの4つでほぼカバーできています。
